# 使用方式

## 1. 建置 CLI

```powershell
cd C:\Users\stw_s\Desktop\ifc_to_3dtiles
cargo build --release
```

輸出：

```text
target\release\ifc_to_3dtiles.exe
```

## 2. 轉 IFC

把 IFC 放在專案根目錄，或把 `--input` 指到實際檔案。

```powershell
.\target\release\ifc_to_3dtiles.exe `
  --input .\DJB-M-SU-_.ifc `
  --output .\out `
  --source-epsg 3826 `
  --tile-max-features 125 `
  --tile-max-triangles 40000 `
  --normal-mode both `
  --smooth-angle-deg 90 `
  --overwrite
```

輸出會在：

```text
out\DJB-M-SU-_
```

不要把 `--output` 指到 `out\DJB-M-SU-_`，程式會自動在 output 底下再建立 IFC 檔名資料夾。

## 2A. 轉 RVT

RVT 需要本機合法 Revit 2025 / 2026 / 2027。先建 Revit bridge：

```powershell
pwsh -NoProfile -ExecutionPolicy Bypass -File .\tools\build_revit_bridge.ps1 -Version 2026 -Configuration Release
```

再執行：

```powershell
.\target\release\ifc_to_3dtiles.exe `
  --input ".\阻尼器(比例需調整).rvt" `
  --output .\out `
  --normal-mode both `
  --revit-version auto `
  --keep-ifc `
  --overwrite
```

`--keep-ifc` 會保留 Revit 匯出的中間 IFC，方便檢查。更完整說明在 `docs\rvt_revit_ifc.md`。

若 console 顯示找不到 Revit，請從官方 Autodesk 入口安裝：

- 已有授權 / 訂閱：`https://manage.autodesk.com/products`
- 試用版：`https://www.autodesk.com/products/revit/free-trial`

若 Revit 裝在非預設路徑，可加：

```powershell
--revit-exe "D:\Autodesk\Revit 2026\Revit.exe" --revit-version 2026
```

## 3. 產出內容

基本輸出：

```text
out\DJB-M-SU-_\DJB-M-SU-_flat.glb
out\DJB-M-SU-_\DJB-M-SU-_smooth.glb
out\DJB-M-SU-_\metadata.json
out\DJB-M-SU-_\unsupported_geometry_report.json
out\DJB-M-SU-_\tileset.json
out\DJB-M-SU-_\tiles\*.b3dm
out\DJB-M-SU-_\conversion_report.json
```

`--normal-mode both --smooth-angle-deg 90` 會另外輸出：

```text
out\DJB-M-SU-_\tileset_smooth.json
out\DJB-M-SU-_\tiles_smooth\*.b3dm
```

目前 viewer 的法線滑桿支援三段：

- `平面`：`tileset.json`
- `90°`：`tileset_smooth_90.json`
- `全平滑`：`tileset_smooth.json`

若要同時保留 90° 與全平滑，可把 90° 的 `tiles_smooth` / `tileset_smooth.json` 另存為 `tiles_smooth_90` / `tileset_smooth_90.json`，再用 `--normal-mode smooth --smooth-angle-deg 180` 產生全平滑版本後放回 `tiles_smooth` / `tileset_smooth.json`。

## 6. Viewer 功能

- `選取`：點選物件看 Batch Table metadata。
- `焦點 ON/OFF`：ON 時被選物件保留實體，其他物件半透明線稿；OFF 時仍可選取。
- `爆炸`：只爆炸目前選取物件。
- `移出`：只移出目前選取物件，方向盤支援 8 方向，中間 `0` 還原。
- `透地`：OFF 時地面/水面遮蔽模型；ON 時可看穿地表。
- `量距` / `量面`：點選量測，右鍵完成。
- `底圖`：EMAP5、Google 航照圖、Google 電子地圖、OSM。
- `法線`：平面、90°、全平滑。
- `畫質`：調整 `viewer.resolutionScale`；自動關閉時可手動固定畫質。
- `自動`：圖台移動中立刻降到 50%，停止後回到 100%。

Three.js 比較頁也支援選取、焦點、黃色高亮、爆炸、移出、量距、量面、法線切換、自動畫質與效能面板；純 Three 版用地面/水面/格網排除底圖變因，MapLibre 版支援 EMAP5、Google 航照圖、Google 電子地圖、OSM。

```powershell

```

Phase 1E 不轉 3D Tiles，只針對 quarantine source 做第二層 QA：

```powershell
cd C:\Users\stw_s\Desktop\ifc_to_3dtiles
pwsh -NoProfile -ExecutionPolicy Bypass -File .\tools\run_phase1e_drilldown.ps1
```

輸出：

```text
out\inspect_tamkang\qa\duplicate_pairs.json
out\inspect_tamkang\qa\entity_outliers.json
out\inspect_tamkang\qa\approved_sources.json
out\inspect_tamkang\qa\rejected_sources.json
```

預設分類：

`entity_outliers.json` 會列出 `管理中心_全.dwg` 的最遠 entity、最大 bbox entity、最大 Z range entity 與可疑 layer。`review_report.html` 也會同步加入 Phase 1E QA summary。

## 8C. Phase 1F Approved Source Publish Skeleton

Phase 1F 只做 publish gate，不做正式 CAD -> 3D Tiles：

```powershell
cd C:\Users\stw_s\Desktop\ifc_to_3dtiles
pwsh -NoProfile -ExecutionPolicy Bypass -File .\tools\run_phase1f_publish_skeleton.ps1
```

輸出：

```text
out\inspect_tamkang\normalized\dwg-12d5f1b6\source_manifest.json
out\inspect_tamkang\publish\sources_manifest.json
out\inspect_tamkang\publish\debug_overlays.json
out\inspect_tamkang\publish\index.html
```

規則：

- `publish\sources_manifest.json` 只吃 `qa\approved_sources.json`
- rejected / needs_review 不會進正式 publish runtime source
- `debug_overlays.json` 保留 rejected / needs_review bbox 與 reason，給 QA viewer 切換顯示
- `normalized\<source-id>\source_manifest.json` 只記 traceability，不複製大型 DXF

開 QA viewer：

```powershell
cd C:\Users\stw_s\Desktop\ifc_to_3dtiles
pwsh -NoProfile -ExecutionPolicy Bypass -File .\tools\run_phase1f_viewer.ps1
```

開：

```text
http://127.0.0.1:8120/index.html
```

`index.html` 會內嵌 `sources_manifest.json` 與 `debug_overlays.json`，所以可直接用 `file://` 開；若 viewer 顯示找不到 Cesium，將 `Cesium-1.141` 放到 `publish\Cesium-1.141`，或用既有專案 server 調整相對路徑。Phase 1F viewer 預設不載 Cesium ion 底圖，避免外部底圖 request failed 造成 render loop 停止。

若看到 IIS / ASP.NET `401.3`，那是 IIS AppPool 帳號沒有 `publish` 目錄 ACL；Phase 1F 建議直接用 `run_phase1f_viewer.ps1` 的 `127.0.0.1` server，不要走 IIS。
